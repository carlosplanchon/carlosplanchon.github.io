Memory Management in Rust: Ownership and Borrowing Explained

Understanding Rust's unique approach to memory safety through ownership, borrowing, and lifetimes.

Rust achieves memory safety without a garbage collector through its ownership system. This article explores the core concepts that make this possible.

The Problem

Traditional memory management approaches have trade-offs:

  • Garbage collectors add runtime overhead
  • Manual memory management is error-prone (buffer overflows, use-after-free, double-free)

Rust solves this at compile time through its ownership system.

Ownership Rules

Rust has three fundamental ownership rules:

  1. Each value has a single owner
  2. When the owner goes out of scope, the value is dropped
  3. There can only be one mutable reference OR multiple immutable references
fn main() {
    let s1 = String::from("hello");  // s1 owns the String
    let s2 = s1;                      // ownership moves to s2
    // println!("{}", s1);            // ERROR: s1 no longer valid
    println!("{}", s2);               // Works fine
}

Borrowing

Borrowing allows references to data without taking ownership:

fn calculate_length(s: &String) -> usize {
    s.len()
}  // s goes out of scope, but doesn't drop the String

fn main() {
    let s1 = String::from("hello");
    let len = calculate_length(&s1);  // borrow s1
    println!("Length of '{}' is {}", s1, len);  // s1 still valid
}

Mutable References

fn main() {
    let mut s = String::from("hello");
    change(&mut s);
    println!("{}", s);  // prints "hello, world"
}

fn change(s: &mut String) {
    s.push_str(", world");
}

Rule: You cannot have a mutable reference while immutable references exist.

The Borrow Checker

The borrow checker enforces these rules at compile time:

fn main() {
    let mut s = String::from("hello");
    
    let r1 = &s;     // immutable borrow
    let r2 = &s;     // another immutable borrow - OK
    // let r3 = &mut s;  // ERROR: can't borrow mutably while immutable borrows exist
    
    println!("{} and {}", r1, r2);
    
    let r3 = &mut s;  // OK - previous immutable borrows are done
    println!("{}", r3);
}

Lifetimes

Lifetimes ensure references are valid for as long as they’re used:

fn longest<'a>(x: &'a str, y: &'a str) -> &'a str {
    if x.len() > y.len() { x } else { y }
}

The 'a lifetime annotation specifies that the returned reference will be valid for as long as both input references.

Why This Matters

  • Zero-cost abstractions: Most checks happen at compile time
  • No runtime overhead: No garbage collector
  • Memory safety guaranteed: Impossible to have dangling references
  • Thread safety: The ownership model prevents data races at compile time

Understanding ownership and borrowing is fundamental to writing idiomatic Rust and leveraging its unique safety guarantees.
